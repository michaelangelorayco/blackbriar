Notes:

You will be given a unique cadet id, and you will be required to append your cadet id in every resource that you will create for this lab activity.

For the instructions below, we use A0 as an example cadet id, your cadet id will be different, be very mindful of this.

Good luck!  


1. On the AWS management console, please check if you are in the Singapore region, if not, please switch your region to Singapore.

 


2. On the AWS management console, navigate to Elastic Beanstalk.

 


3. On the top right of the page, click on create new application.

 


4. Name your application webapp followed by your cadet id separated by a dash, then create the tag cadetid and lab.

Set the cadetid value with your assigned cadet id, and just input 4 for the lab value.

Click Create.
 


5. At the top of the web page, find the application you have created and click on create environment.

 


6. Select web server environment.

 


7. On the base configuration, select PHP for the platform.

 
On the application code, select upload your code then click the upload button.
 


8. The source code origin will be Public S3 URL. use this URL: https:https://apper-demo.s3-ap-southeast-1.amazonaws.com/beanstalk-demo/apper.ph-lab4-php-app-11242018.zip

Click upload.
 


9. Click on create configure more option.



10. On the configuration preset, select custom configuration.

Then click modify on capacity.
 


11. Set the minimum instance value to 2.

 
Scroll down.


12. For the scaling triggers, use the same exact values that are defined on the image below:
Click save, then create environment.
 


13. Wait for it to provision the servers.


Once the provisioning is finished you will be redirected to this page:

Take note of the environment URL.
 


14. Navigate to EC2 console.

 


15. Find the instances that were created by elastic beanstalk, look for the one which includes your cadet id in the name.

 


16. Select any of your instances, then terminate it by clicking on actions.

 


17. Click on Yes, Terminate

Your instance will be terminated, wait for a few minutes.
 


18. After your instance has been terminated you'll see that a new server will be created automatically, in replacement of the server you have terminated.
