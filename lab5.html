Notes:


This will be a group activity, there are 5 CloudFormation templates and you can assign one to each member of the group.

Here are the links for the CloudFormation templates:

A. IAM: https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/IAM_CFN.json

B. EC2: https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/EC2_CFN.json

C. S3: https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/S3_CFN.json

D. DynamoDB: https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/DYNAMODB_CFN.json

E. Nested Stack: https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/NESTED_STACK.json


Objective:

This activity will require you to modify the CloudFormation template to add output. The nested stack will be dependent on the IAM, EC2, S3 and DynamoDB stack. If you are assigned on the Nested Stack, you have to wait for the IAM, EC2, S3 and DynamoDB stack to finish.

Also, your team needs to create a public S3 bucket to store the updated IAM, EC2, S3, DynamoDB and Nested Stack templates, make sure to make the objects public as well so that CloudFormation will be able to access it.

Name your S3 bucket lab5 followed by your team id separated by a dash.



Below are the specific guides for each stack:

A. IAM Stack.


A1. Make sure you are in the Singapore region.


A2. Navigate to CloudFormation console.


A3. Click on create stack.






A4. Use this URL https://apper-demo.s3-ap-southeast-1.amazonaws.com/cf-templates-lab-5/IAM_CFN.json as the template URL.



Click next.




A5. Set the name IAM-STACK.



Click next.




A6. For this lab we don't need to configure the tags, so just leave it blank.

Scroll down and click next.




A7. Acknowledge that it will create resources and click create stack.

 




A8. Once the stack creation is complete, click on the outputs tab and you will see the AccessKey and SecretKey output.

 




A9. Here comes the work, you need to output the username of the IAM User, you can do this by opening the IAM CloudFormation template URL in your web browser, copy the text and then paste it into a file in your local machine and save it with a .json file extension.

Then you need to modify the output section of the CloudFormation template.

After you have made the modification, you can test it by clicking on update which is located at the top of the IAM-STACK you have created.

   




A10. In the prerequisite select "replace current template", then select "upload a template file" for the template source, click choose file and upload the CloudFormation template from your local machine.  



Click next.  




A11. Leave the parameters as is.



Click next.  




A12. Leave the configure stack options as is.



Scroll down and click next.  




A13. Acknowledge that it will create resources then click update stack.

 




A14. Once the update stack is complete, you should see that the username is now included in the output.



Your updated S3 CloudFormation template can now be uploaded in the public S3 bucket.
